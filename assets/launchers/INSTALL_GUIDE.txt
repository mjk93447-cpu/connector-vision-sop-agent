================================================================
 Connector Vision SOP Agent v4.2.0 - Split Deployment Package
 INSTALL GUIDE
================================================================

PACKAGE ARTIFACTS
--------------------------------------------------
  1. connector-agent-app.zip
     - connector_vision_agent.exe
     - launchers and config/model placeholders
  2. connector-agent-pretrain.zip
     - connector_pretrain.exe
     - pretrain_data\ with the bundled pcb_inspection + pcb_component_detection + pcb_defect_detection + rf100_smd_components + rf100_deeppcb dataset
  3. connector-agent-llm.zip
     - ollama_models\blobs\
     - ollama_models\manifests\

ASSEMBLY
--------------------------------------------------
  1. Extract connector-agent-app.zip into the final deployment folder.
  2. Extract connector-agent-pretrain.zip into the same folder.
  3. Extract connector-agent-llm.zip into ollama_models\.
  4. Replace PLACE_* txt files with the real artifact contents.

RUN
--------------------------------------------------
  - Double-click start_agent.bat to launch the main SOP agent.
  - Double-click start_pretrain.bat after the pretrain artifact is merged.
  - Keep ollama_models\ separate if you only want to refresh the LLM blobs.

FINAL FOLDER STRUCTURE
--------------------------------------------------
  connector_agent\
  +-- connector_vision_agent.exe
  +-- connector_pretrain.exe
  +-- start_agent.bat
  +-- start_pretrain.bat
  +-- stop_ollama.bat
  +-- assets\
  |   +-- config.json
  |   +-- sop_steps.json
  |   +-- models\
  |       +-- yolo26x.pt
  |       +-- yolo26x_local_pretrained.pt
  +-- pretrain_data\
  +-- ollama_models\
  +-- MERGE_GUIDE.txt
  +-- PLACE_APP_HERE.txt
  +-- PLACE_PRETRAIN_DATA_HERE.txt
  +-- PLACE_LLM_HERE.txt

NOTES
--------------------------------------------------
  - The app bundle is intentionally lean and does not ship the LLM blobs.
  - The pretrain bundle is a separate temporary artifact and can be refreshed
    independently of the main app pack.
  - The LLM bundle only needs rebuilding when OCR/LLM behavior changes.
