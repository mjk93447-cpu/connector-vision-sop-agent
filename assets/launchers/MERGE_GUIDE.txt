================================================================
Connector Vision SOP Agent v6.0.0 - Deployment Guide
================================================================

REQUIRED DOWNLOADS
  1. connector-agent-app-cpu.zip   full app pack with CPU-only torch runtime
  2. connector-agent-app-gpu.zip   full app pack with CUDA-enabled torch runtime

OPTIONAL DOWNLOADS
  3. connector-agent-llm-local-cache.zip       local Ollama model cache
     or split parts connector-agent-llm-local-cache.zip.001 / .002 ...

CHOOSE ONE APP PACK
  - CPU pack:
    normal on CPU-only PCs
    normal on GPU PCs, but CPU only

  - GPU pack:
    CUDA preferred on NVIDIA PCs
    still usable on CPU-only PCs through CPU fallback

INSTALL STEPS
  1. Create folder: C:\tools\connector_agent\
  2. Extract exactly one app pack into that folder:
     - connector-agent-app-cpu.zip
       or
     - connector-agent-app-gpu.zip
  3. Optional: extract connector-agent-llm-local-cache.zip into
     C:\tools\connector_agent\ollama_models\
     If the artifact was split, join the `.zip.001` / `.zip.002` parts first.

FINAL FOLDER STRUCTURE
  connector_agent\
  +-- connector_vision_agent.exe   <- SOP agent with bundled runtime
  +-- start_agent.bat              <- DOUBLE-CLICK TO RUN
  +-- runtime_flavor.txt
  +-- assets\
  |   +-- config.json
  |   +-- sop_steps.json
  |   +-- models\
  |       +-- yolo26x.pt
  |       +-- yolo26x_pretrained.pt
  |       +-- yolo26x_local_pretrained.pt
  +-- stop_ollama.bat
  +-- ollama_models\
      +-- blobs\sha256-...
      +-- manifests\...

FEATURE PARITY
  CPU and GPU app packs share the same code and user-facing features:
  - GUI
  - fine-tuning
  - SOP Run
  - SOP Editor
  - Config
  - Audit
  - SOP Generate

  The only intended runtime difference is torch acceleration behavior.
  The runtime itself is embedded into the EXE instead of shipping as a
  separate `_internal` folder.

HOW TO RUN
  Double-click start_agent.bat
  -> Starts Ollama server when available
  -> Opens the 7-tab PyQt6 GUI

CUSTOMIZATION
  - Edit assets\config.json to tune timing, thresholds, and runtime behavior
  - Edit assets\sop_steps.json to modify SOP steps without rebuilding
  - Use Tab 4 SOP Editor for live SOP updates

LLM MODEL INFO
  Target deployment: Gemma 4 26B A4B Q4_K_M + TurboQuant
  Packaged cache    : local Ollama blobs/manifests copied from the build machine
  Split policy      : individual model blobs larger than 2 GB are chunked for transport

TROUBLESHOOTING
  - If GUI does not open: run start_agent.bat from cmd and inspect the output
  - If SOP Generate fails: ensure ollama.exe and the expected model cache are available
  - If YOLO detects nothing: verify assets\models\yolo26x_local_pretrained.pt
