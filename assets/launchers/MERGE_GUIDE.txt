================================================================
Connector Vision SOP Agent v4.4.0 - Portable Bundle Guide
================================================================

DOWNLOADS REQUIRED (GitHub Actions -> Artifacts tab):
  1. connector-agent-app.zip         app EXE + launchers + config/models
  2. connector-agent-pretrain.zip    local pretrain EXE + dataset
  3. connector-agent-llm.zip         IBM Granite Vision 3.2-2b blobs

ASSEMBLY STEPS:
  1. Create folder: C:\tools\connector_agent\
  2. Extract connector-agent-app.zip
     -> all files go into C:\tools\connector_agent\
  3. Extract connector-agent-pretrain.zip
     -> all files go into C:\tools\connector_agent\pretrain_data\ and the EXE
  4. Extract connector-agent-llm.zip
     -> all files go into C:\tools\connector_agent\ollama_models\
        (blobs\ and manifests\ must be directly inside ollama_models\)

FINAL FOLDER STRUCTURE:
  connector_agent\
  +-- connector_vision_agent.exe   <- SOP agent (YOLO26x + PyQt6 GUI embedded)
  +-- start_agent.bat              <- DOUBLE-CLICK TO RUN (GUI mode)
  +-- connector_pretrain.exe       <- Local pretrain runner (epochs/batch only)
  +-- assets\
  |   +-- config.json              <- settings (editable)
  |   +-- sop_steps.json           <- 40-step atomic SOP definition (editable)
  |   +-- models\
  |       +-- yolo26x.pt           <- YOLO weights
  |       +-- yolo26x_pretrain.pt  <- cloud/GitHub pretrain checkpoint
  |       +-- yolo26x_local_pretrained.pt <- offline pretrain output
  +-- start_pretrain.bat           <- DOUBLE-CLICK TO START LOCAL TRAINING
  +-- stop_ollama.bat
  +-- pretrain_data\               <- optional local dataset folder (not bundled in GitHub artifact)
  |   +-- source data: pcb_inspection + pcb_component_detection + pcb_defect_detection + rf100_smd_components + rf100_deeppcb
  +-- ollama_models\
      +-- blobs\sha256-...         <- from connector-agent-llm zip
      +-- manifests\...            <- from connector-agent-llm zip
  +-- PLACE_APP_HERE.txt           <- root deployment note
  +-- pretrain_data\PLACE_PRETRAIN_DATA_HERE.txt
  +-- ollama_models\PLACE_LLM_HERE.txt

MERGE ORDER:
  1. Extract connector-agent-app.zip into the root folder.
  2. Extract connector-agent-pretrain.zip into the same root folder.
  3. Extract connector-agent-llm.zip into ollama_models\.
  4. Verify the placeholder txt files were replaced by real content.

HOW TO RUN:
  Double-click start_agent.bat
  -> Starts Ollama server (LLM backend)
  -> Opens PyQt6 GUI with 7 tabs:
       Tab 1: Run SOP    - 40-step atomic SOP execution with live log
       Tab 2: Vision     - Screenshot capture + YOLO bbox overlay
       Tab 3: LLM Chat   - ROI screenshot + ChatGPT-like streaming + Stop button
       Tab 4: SOP Editor - Edit/reorder SOP steps (type_text/press_key/wait_ms)
       Tab 5: Config     - Edit config.json via UI
       Tab 6: Audit      - Config change history
       Tab 7: Training   - Annotate connector images and fine-tune YOLO26x

  Double-click start_pretrain.bat
  -> Starts local YOLO26x pretraining on your local pretrain_data\ folder
  -> Uses the local GPU when available
  -> Produces assets\models\yolo26x_local_pretrained.pt

CANONICAL PATHS
--------------------------------------------------
  - Primary pretrain entrypoint: scripts/run_pretrain_local.py
  - Legacy pretrain scripts are deprecated and kept only for compatibility
  - Model naming reference: docs/MODEL_ARTIFACT_NAMING.md

CUSTOMIZATION:
  - Edit assets\config.json to tune pin counts, timing, thresholds
  - Edit assets\sop_steps.json to modify the 40-step atomic SOP sequence
    (changes take effect immediately without rebuilding EXE)
  - Tab 4 SOP Editor: change PW, key bindings, wait duration directly in GUI

LLM MODEL INFO:
  Name  : IBM Granite Vision 3.2-2b (ibm/granite3.2-vision:2b)
  Vendor: IBM Corporation (USA) - non-Chinese model
  Size  : ~2.0 GB (fully offline, multimodal)
  Notes : Vision + text, SOP log analysis, ROI screenshot analysis

TROUBLESHOOTING:
  - If GUI does not open: check Windows Event Log or run from cmd prompt
  - If LLM chat fails: ensure ollama.exe is running (check Task Manager)
  - If YOLO detects nothing: yolo26x.pt may be a placeholder (CI build)
